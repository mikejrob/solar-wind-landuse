# Writing style for this repository

All prose in this repository follows these rules. Apply them to notes, docs,
the README, and the paper.

## Language

- Lead with the answer. Put the finding in the first sentence.
- Write short declarative sentences.
- Do not use filler words: "structurally," "honestly," "legitimately,"
  "essentially," "fundamentally," "notably," "importantly," "arguably."
- Do not use binary framing: "not X but Y," "X, not Y," "instead of X, Y,"
  "rather than." State what is.
- Do not use meta-commentary or padding: "it is important to note," "let's,"
  "the key point is," "worth flagging."
- Do not use flowery adjectives or rhetorical build-up.

## Evidence

- Every factual claim ends with a source citation (URL or repository path).
- Claims that cannot be traced to a source are flagged `UNVERIFIED` or omitted.
- Numbers, dates, and legal citations are verified against a primary source.
- Report what the record shows. Report null results.
